# Weekly Summary Templates

下面提供两个模板，默认优先使用“简洁版”。如果项目多、push 较多、还需要交代阶段状态，再切换成“复杂版”。

## 1. 简洁版

```md
# 本周工作周报

## 本周工作概览
- 本周主要参与了 `项目A` 和 `项目B` 的开发/排查工作，重点在于：`一句话概括重点`
- 目前已经完成：`已完成内容`
- 仍在跟进：`未完成内容`

## 按项目进展

### 项目A
- `05-12 Push 1`：完成了 `动作`，当前结果是 `结果/状态`
- `05-14 Push 2`：补充了 `动作`，解决/推进了 `问题`
- 辅助记录：本周 Codex 对话主要围绕 `背景/问题/方案`，可作为本次修改的上下文

### 项目B
- `05-13 Push 1`：协助处理 `任务`
- `05-15 Push 2`：完成 `任务` 的基础部分，目前 `状态`

## 个人思考
- 这周在 `某个任务` 上，感受到自己还需要继续补强 `技术点/业务理解/沟通方式`
- 通过这次 `开发/排查/联调`，对 `某个模块或流程` 的理解更清楚了一些，后面希望继续补上 `具体方向`
- 下周除了完成手头任务，也想继续熟悉 `代码结构/工具链/测试流程`

## 下周计划
- 完成 `任务1`
- 跟进 `任务2`
- 根据联调/测试结果继续修正 `任务3`
```

## 2. 复杂版

```md
# 本周工作周报

## 本周工作概览
- 本周主要围绕多个项目做了功能开发、问题排查和联调支持，重点集中在 `方向A`、`方向B` 和 `方向C`。
- 已完成：`本周已落地的主要结果`
- 当前进度：`目前推进到哪一步`

## 按项目进展

### 项目A
- 项目背景：`一句话说明这个项目/模块本周在做什么`
- `05-12 Push 1`：完成 `动作A`，产出是 `结果A`，目前 `状态A`
- `05-14 Push 2`：修复 `问题B`，推进了 `结果B`，目前 `状态B`
- 本周补充：Codex 对话主要补充了 `问题背景`、`排查过程` 和 `方案取舍`
- 当前状态：`这一项目现在整体到哪一步`

### 项目B
- 项目背景：`一句话背景`
- `05-13 Push 1`：协助处理 `动作C`，当前 `状态C`
- `05-15 Push 2`：完成 `动作D` 的基础部分，目前 `状态D`
- 当前状态：`后续还需要继续跟进什么`

## 个人思考
- `这周比较值得复盘的一点`
- `后面希望继续加强的方向`

## 下周计划
- `计划1`
- `计划2`
```

## 3. 句式参考

推荐句式：

- `完成了 xxx 的基础开发，当前已经可以 xxx。`
- `协助排查 xxx 问题，初步确认原因与 xxx 有关。`
- `补充了 xxx 逻辑，推进到 xxx 阶段。`
- `根据 Codex 对话和提交记录，主要工作集中在 xxx。`
- `这周在 xxx 过程中，感觉自己还需要继续熟悉 xxx。`
- `通过这次任务，对 xxx 的理解比之前更具体了一些。`
- `后面希望继续补上 xxx 这部分能力。`

避免句式：

- `深入推进相关工作`
- `全面完成多项优化`
- `取得显著成果`
- `本周收获很多，能力得到了全面提升`
- `在老师指导下取得了巨大进步`

## 4. 使用提醒

- 如果拿不到真实 push 记录，把标题改成 `提交批次 1 / 2 / 3`
- 如果某项目本周只有零散小改动，可以合并成一个批次写
- 如果只有对话没有提交，只能写成“调研/排查/讨论”，不能写成“完成开发”
- 个人思考默认由用户自己补充，AI 更适合提供角度、提纲和轻量草稿
- 面向用户的说明性备注，例如取证缺口、工具限制、执行过程偏差，不要写进最终提交版周报
- 用户已经给了明确目标路径时，直接写入目标文件，不再额外在 skill 目录保留同名草稿
